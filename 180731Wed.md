이탈 예측을 하는 이유 : 유저의 이탈을 방지해 고객 생애 가치(LTV), 투자 대비 수익(ROI)를 정확히 계산해 데이터 기반의 의사 결정에 일조

이탈 예측 모델링 프로세스 : 1) 학습 데이터 생성, 2) 데이터 가공, 3) 모델 생성, 4)성능 평가
1) 학습 데이터 생성
이탈의 정의 : 레이블로 정의, 대체로 이탈자는 장기 미접속 유저, 기간을 잘 정해야
이탈 예측 대상 결정 : 봇, 악성 유저, 친구 따라 하는 유저를 제외한 진성 유저 + 

2) 데이터 가공
어떤 걸로 이탈자 판단할지 -> 이탈자는 활동이력 기준으로 플레이 시간이 줄어드는 경우가 많고, 사회활동 기준으로 클랜/파티/길드 등에 가입하고 탈퇴하는 경우 많음

3) 모델 생성
어떤 알고리즘으로 예측할지
- 예측력과 해석력 : 예측력은 좋은데 왜 유저가 이탈하는지는 모르는 알고리즘, 예측력은 낮지만 왜 이탈하는지는 정확한 알고리즘, 그리고 예측력과 해석력이 적당한 분배를 이룬 알고리즘 -> 한 쪽만 높으면 다른 알고리즘을 또 사용해야 하므로 적당한 분배 중요
- 예측 여부와 이탈 기간 : 이탈 기간을 알려주는 알고리즘 추천

4) 성능 평가
확실히 이탈 예측 가능한 유저만 예측할지, 최대한 많은 유저들을 잠재 이탈자로 간주할 지 고려해야 + 정확히 예측했는지 예의주시
출첵만 하는 유저, 내일 이탈한다고 예측한 유저가 3일 후에 이탈해서 애매한 경우야 Time Dependent ROC Curve나 IBS(Integerated Brie Score) 등 다수의 평가 데이터를 사용할 필요
